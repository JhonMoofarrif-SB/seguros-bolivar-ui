Instalación

Existen tres formas principales de usar Seguros Bolívar UI en tu proyecto. Elige la que mejor se adapte a tus necesidades:

1. CDN (Recomendado para Prototipos)

La forma más rápida de empezar. Simplemente incluye los archivos CSS y JS desde un CDN público:

unpkg.com

<!DOCTYPE html> <html lang="es"> <head> <!-- CSS del Design System --> <link rel="stylesheet" href="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-seguros-bolivar-light.min.css"> <!-- Web Components (opcional) --> <script type="module" src="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-components.min.js"></script> </head> <body> <button class="sb-ui-button sb-ui-button--primary sb-ui-button--fill"> ¡Hola Mundo! </button> </body> </html>

jsDelivr (Alternativa)

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-seguros-bolivar-light.min.css"> <script type="module" src="https://cdn.jsdelivr.net/npm/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-components.min.js"></script>
Ventajas del CDN:
  • ✅ Configuración instantánea - sin instalación
  • ✅ Caché global - carga más rápida
  • ✅ Ideal para prototipos y demos
  • ✅ Sin gestión de dependencias

2. NPM (Recomendado para Producción)

Para proyectos en producción, instala el paquete como una dependencia:

# NPM npm install @seguros-bolivar/ui-bundle # PNPM pnpm add @seguros-bolivar/ui-bundle # Yarn yarn add @seguros-bolivar/ui-bundle

Luego importa en tu aplicación:

// En tu archivo principal (ej: main.js, index.js) import '@seguros-bolivar/ui-bundle/dist/sb-ui-seguros-bolivar-light.min.css'; import '@seguros-bolivar/ui-bundle/dist/sb-ui-components.min.js';
Ventajas de NPM:
  • ✅ Control de versiones preciso
  • ✅ Optimización con bundler (Webpack, Vite, etc.)
  • ✅ Tree-shaking disponible
  • ✅ Integración con sistemas de build

Nomenclatura BEM

Todos los componentes siguen la metodología BEM (Block, Element, Modifier) con el prefijo sb-ui-:

Patrón Ejemplo Descripción
.sb-ui-{block} .sb-ui-button Bloque base del componente
.sb-ui-{block}--{modifier} .sb-ui-button--primary Modificador de variante
.sb-ui-{block}--{size} .sb-ui-button--small Modificador de tamaño
.sb-ui-{block}--{state} .sb-ui-button--disabled Modificador de estado
¿Por qué BEM?

BEM hace que el CSS sea predecible, reutilizable y escalable. El prefijo sb-ui- evita conflictos con otros estilos de tu proyecto.

Cambiar de Marca

Para cambiar de marca, simplemente cambia el archivo CSS que importas:

<!-- Seguros Bolívar Light --> <link rel="stylesheet" href="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-seguros-bolivar-light.min.css"> <!-- Davivienda Dark --> <link rel="stylesheet" href="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-davivienda-dark.min.css"> <!-- Jelpit Light --> <link rel="stylesheet" href="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-jelpit-light.min.css">

Cambiar de Tema Dinámicamente

Puedes permitir a los usuarios cambiar entre tema claro y oscuro en tiempo real:

// JavaScript para cambiar tema dinámicamente function cambiarTema(marca, tema) { const linkCSS = document.querySelector('link[rel="stylesheet"]'); const nuevoCSS = `https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-${marca}-${tema}.min.css`; linkCSS.href = nuevoCSS; } // Uso: cambiarTema('seguros-bolivar', 'dark'); // Cambia a tema oscuro cambiarTema('davivienda', 'light'); // Cambia a Davivienda light

Prueba Cambiar de Marca:

¿Qué incluye cada marca?
  • Tokens: Colores, tipografía, espaciado personalizados
  • Componentes: Todos los componentes con estilos de la marca
  • Overrides: Personalizaciones específicas por marca
  • Todo en uno: Un solo archivo CSS contiene todo

Personalización

Variables CSS

Todos los componentes usan CSS Variables (Custom Properties) que puedes sobrescribir:

/* Sobrescribir colores principales */ :root { --sb-ui-color-primary-base: #your-color; --sb-ui-color-secondary-base: #your-color; } /* Sobrescribir espaciado de un componente específico */ .sb-ui-button { --sb-ui-button-padding-inline: 2rem; --sb-ui-button-padding-block: 1rem; --sb-ui-button-border-radius: 8px; }

Tokens Principales

Token Descripción Ejemplo
--sb-ui-color-primary-base Color primario de la marca #038450 (Seguros Bolívar)
--sb-ui-color-secondary-base Color secundario de la marca #FFE16F (Seguros Bolívar)
--sb-ui-typography-fontFamily Familia tipográfica 'Roboto', sans-serif
--sb-ui-spacing-base Espaciado base 1rem

Crear Tu Propia Marca

Puedes extender el Design System creando tus propios brand overrides:

/* mi-marca.css */ :root { /* Colores */ --sb-ui-color-primary-base: #1e88e5; --sb-ui-color-secondary-base: #ffa726; /* Tipografía */ --sb-ui-typography-fontFamily: 'Open Sans', sans-serif; /* Espaciado */ --sb-ui-button-padding-inline: 2rem; --sb-ui-button-border-radius: 12px; } /* Personalización específica de componentes */ .sb-ui-button--primary { --sb-ui-button-bg: var(--sb-ui-color-primary-base); --sb-ui-button-text: white; }

Luego carga tu CSS personalizado después del CSS del Design System:

<link rel="stylesheet" href="https://unpkg.com/@seguros-bolivar/ui-bundle@latest/dist/sb-ui-white-label-light.min.css"> <link rel="stylesheet" href="mi-marca.css">

Mejores Prácticas

Hacer (Do's)

  • ✅ Usa las clases CSS directamente - No crees wrappers innecesarios
  • ✅ Combina modificadores - .sb-ui-button--primary.sb-ui-button--small
  • ✅ Usa variables CSS - Para personalización consistente
  • ✅ Sigue BEM - Respeta la nomenclatura del sistema
  • ✅ Prueba accesibilidad - Usa lectores de pantalla
  • ✅ Mobile-first - Diseña primero para móvil
  • ✅ Usa CDN para prototipos - NPM para producción
  • ✅ Versiona tu CSS - Usa @1.0.0 en lugar de @latest en producción

No Hacer (Don'ts)

  • ❌ No sobrescribas con !important - Usa CSS Variables
  • ❌ No modifiques las clases base - Crea nuevos modificadores
  • ❌ No mezcles estilos - Usa solo el Design System
  • ❌ No uses IDs para estilos - Solo clases
  • ❌ No ignores la accesibilidad - Siempre usa atributos ARIA
  • ❌ No uses @latest en producción - Puede romper tu app
  • ❌ No cargues múltiples marcas - Solo una por página
